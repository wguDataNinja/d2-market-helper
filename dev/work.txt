5️⃣ 🔬 Normalization Logic

item_name	item_id	ist_value	confidence_flag	last_updated

7️⃣ 🩺 The Bulk Distortion Problem

⸻

✅ Problem Definition

Certain trades reflect bulk listings which get incorrectly interpreted by the processor.

Example:

{
  "quantity": 20,
  "price": [{ "name": "Ber Rune", "quantity": 1 }]
}

Seller’s intent:
Selling 20 Jah Runes for 1 Ber Rune each.

Processor output:
Treats as Jah : Ber = 20:1 (incorrect — huge outlier).

⸻

✅ How This Happens
	•	Current processor uses:

ratio = quantity : price[].quantity


	•	It assumes the entire quantity applies to the price, without recognizing unit pricing intent.
	•	This creates extreme ratio outliers that inflate FMV calculations if not filtered.

⸻

✅ Where This Applies

Item Type	Bulk Distortion Seen?
Runes	✅ Frequently
Gems	✅ Frequently
Misc	✅ Occasionally


⸻

✅ Known Bulk Patterns
	•	Rune-for-rune offers with bulk quantity (quantity > 1).
	•	Rune-for-gems offers where gems offered as bulk (price[].quantity >> 1).
	•	Multi-item bundles where some price components are multiples.

⸻

8️⃣ 🔬 Current Bulk Filtering Approach

✅ Outlier rejection:
	•	After initial Ist-normalization:
	•	Drop any trades where:

implied price > 2 × normalized price


	•	This removes most extreme bulk distortions.
