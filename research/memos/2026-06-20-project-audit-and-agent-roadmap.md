## Executive Summary

D2R Market Helper is currently best described as:

> Traderie-normalized in-game rune values, separate external cash comparison, and a source-transparency ledger.

The honest production model source is still Traderie only. Diablo2.io has a parser prototype and fixture evidence, but the sold-search volume is too thin for modeling. External cash sources are correctly separate from in-game values and every cash observation in the merged product has `use_in_model=false`.

The repository is not clean. Several important milestone docs and research artifacts are untracked, while core files are modified. The largest operational inconsistency is documentation drift: `docs/PROJECT_MEMORY.md` mostly matches the current repo state, but `docs/COLLECTION_RUNBOOK.md`, `docs/USERSCRIPT.md`, `docs/DATA_PRODUCTS.md`, `docs/PRICING_MODEL.md`, `docs/SOURCE_MANIFEST.md`, and the userscript feed product are not fully reconciled with the current generator/script state.

No launchd state was inspected or mutated during this audit. Operational truth is based only on repo docs/scripts and ignored runtime files visible in the workspace.

- Source confidence is not medium-high yet. The completed-trade source discovery plan has not been executed.
