# Traderie Userscript Data Feed

## Overview

The D2R Market Helper project produces the pricing data consumed by the **Traderie Tools** Tampermonkey userscript. The userscript lives at a separate GitHub repo (`github.com/wguDataNinja/TraderieTools`) and fetches prices from this project's data feed.

## Data Feed Files

| File | Format | Purpose |
|---|---|---|
| `data/products/traderie_tools_prices.json` | v0.2 structured with `segments` key | Canonical feed for the userscript v2+ |
| `data/products/rune_prices_legacy.json` | Flat per-segment keys (no wrapper) | Legacy compatibility for existing userscript installations |

Both files are generated by `scripts/generate_prices_json.py`.

### Legacy Format (`rune_prices_legacy.json`)

```json
{
  "pc_sc_nl": {
    "Ber Rune": {
      "ist_value": 12.43,
      "low_confidence": false,
      "confidence": "high",
      "bid_price": 11.85,
      "ask_price": 12.99,
      "total_trades": 220
    },
    "Jah Rune": { ... }
  },
  "pc_sc_l": { ... },
  "pc_hc_l": { ... },
  "pc_hc_nl": { ... }
}
```

- Segment keys match `getServerSlug()` output: `pc_sc_l`, `pc_sc_nl`, `pc_hc_l`, `pc_hc_nl`
- Rune names use `"Ber Rune"` format (with " Rune" suffix)
- `ist_value`: Blended VWAP in Ist equivalent
- `low_confidence`: Boolean — `true` if confidence is "low" or "unavailable"
- `confidence`: String — "high", "medium", "low", or "unavailable"
- `bid_price`: Average Ist offered by buyers
- `ask_price`: Average Ist requested by sellers
- `total_trades`: Number of completed trades used in the model

### Structured Format (`traderie_tools_prices.json`)

```json
{
  "schema_version": "0.2",
  "generated_at": "2026-06-20T12:00:00Z",
  "last_update": "2026-06-20T04:14:51Z",
  "segments": {
    "pc_sc_nl": { "Ber Rune": { ... } }
  }
}
```

## Segment Detection

The userscript detects the current segment from the Traderie page URL:

```javascript
function getServerSlug() {
  const p = new URLSearchParams(window.location.search);
  const plat = (p.get('prop_Platform') || 'pc').toLowerCase();
  const mode = p.get('prop_Mode') === 'hardcore' ? 'hc' : 'sc';
  const lad = p.get('prop_Ladder') === 'true' ? 'l' : 'nl';
  return `${plat}_${mode}_${lad}`;
}
```

If no URL filters are set, defaults to `pc_sc_nl`. A caveat should be displayed when using the default.

## Evaluation Rules

The overlay evaluates trades using these rules:

| Condition | Label | Visual |
|---|---|---|
| Receiver gains ≥ +0.5 Ist over ask | Good deal | Green |
| Within ±0.5 Ist of blended VWAP | Fair range | White/grey |
| Payer loses ≥ -0.5 Ist below bid | Overpay | Red |
| Rune confidence is "low" or "unavailable" | Low confidence | Orange badge |
| Listing requests multiple items (AND trade) | Complex — review manually | Yellow badge |
| No price data for this rune/segment | Unknown | Grey |

### Rules
- **Cash/RMT prices are never used** in the overlay. The overlay evaluates in-game trades only.
- **Active listings are not completed-trade evidence.** Only completed trades from the Traderie API are used.
- **Segment mismatch must warn.** If the detected segment has no data for a rune, show "unavailable" rather than falling back to a different segment.
- **AND/bundle trades** should show "Complex — review manually" unless reliable parsing is confirmed.

## Safety Rules

- The userscript is **read-only**. It does not:
  - Automate listing creation or acceptance
  - Modify account data
  - Submit forms
  - Login or store credentials
- The data feed contains **in-game barter values only**. No cash/RMT prices.
- Prices are **relative to Ist** — not absolute USD values.

## Methodology

- **Source:** Traderie completed trades only (unofficial API)
- **Window:** Rolling 50-cap recent trades per item/segment
- **Model:** Ist-normalized bid/ask VWAP
- **Excluded:** AND trades, non-Ist pairs, active listings, cash prices, Reddit/community data
- **Confidence:** High (50+ trades), Medium (15-49), Low (1-14), Unavailable (0)

## Known Limitations

- Traderie API behavior is unofficial and may change
- Thin-volume segments (Hardcore Non-Ladder) have mostly unavailable prices
- AND trades and non-Ist pairs are excluded from the model
- Prices are relative in-game values, not absolute cash equivalents
- The rolling 50-cap means prices reflect recent market activity only

## Deploying the Feed

1. Regenerate products: `python3 scripts/generate_prices_json.py`
2. Copy `data/products/rune_prices_legacy.json` to the userscript repo as `rune_prices.json`
3. Copy `data/products/traderie_tools_prices.json` to the userscript repo
4. Or configure the userscript to fetch directly from this repo's raw GitHub URL

## Fallback Strategy

When the data feed fails to load:
1. Use cached prices from `localStorage` (1-hour TTL)
2. Show "(cached)" indicator when using stale data
3. If no cache exists, disable pricing module and show "Pricing temporarily unavailable"

## Cache Implementation

```javascript
const CACHE_KEY = 'traderie-rune-prices-cache';
const CACHE_TTL_MS = 3600000; // 1 hour

function getCachedPrices() {
  const cached = localStorage.getItem(CACHE_KEY);
  if (!cached) return null;
  const { prices, timestamp } = JSON.parse(cached);
  if (Date.now() - timestamp > CACHE_TTL_MS) return null;
  return prices;
}

function setCachedPrices(prices) {
  localStorage.setItem(CACHE_KEY, JSON.stringify({ prices, timestamp: Date.now() }));
}
```
