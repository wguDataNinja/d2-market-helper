## 3. Relationship between repos

`TraderieTools` is a separate, independent userscript repository. It:

- Consumes `rune_prices.json` from the `TraderieTools` GitHub raw URL at runtime.
- Does not depend on the `traderie` repository for build or deployment.
- Does not share credentials, database connections, or Git history.

The `traderie` repo generates product JSONs (`data/products/`) that are published independently. The userscript currently does not consume those product files; it uses `rune_prices.json` stored in its own repo.

---

## 4. VPS deployment pinning

Future VPS deployment of the `traderie` repo must:

1. Clone from `git@github.com:wguDataNinja/d2-market-helper.git` at the approved SHA.
2. Verify SHA against the registry (`repos/traderie/CONTROL.md` §Approved SHA).
3. Confirm clean checkout: `git status --porcelain` returns no modified tracked files.
4. Report `deployed_revision` in health payload.
5. Detect drift by comparing `git rev-parse HEAD` against the registered approved SHA.

`TraderieTools` is never deployed to VPS. It is a browser-side userscript installed via Tampermonkey/Greasymonkey. Its canonical version is the `@version` metadata in `traderie-tools.user.js` (currently `2026-06-26`). It is published to users via GreasyFork (not yet set up) or direct `raw.githubusercontent.com` URL.

---

## 5. Drift detection after deployment

Drift is detected by comparing:

- `git rev-parse HEAD` in the deployed checkout against the approved SHA in `repos/traderie/CONTROL.md`.
- Health payload `deployed_revision` field against the registry.
- Dirty checkout: `git status --porcelain` must be empty.

Detection is read-only. No automatic correction. Drift findings are reported through health output.
